# ADR 0005: Портфолио-артефакт, а не личный инструмент

**Статус:** принято
**Дата:** 2026-07-15

## Контекст

Дефолт для пет-проекта — минимально полезная вещь: строишь для себя, запускаешь в терминале, без
auth, без деплоя, без CI, пока инструмент не докажет свою полезность. Это здравый дефолт, и часть
его здесь *намеренно не соблюдается*. Отклонения нужно зафиксировать, чтобы будущий читатель (или
сам автор) не «упростил» репозиторий обратно до бесполезности для его настоящей цели.

## Решение

GH Pulse оптимизируется под другую цель, чем личный инструмент: **это артефакт, который читают.**
Его пользователь — не автор, а инженер, который потратит на репозиторий ~5 минут и захочет понять,
как устроена и почему быстро работает data-heavy система. Эта цель переворачивает несколько
привычных дефолтов:

| Дефолт пет-проекта | Здесь | Почему отклонение верное |
|---|---|---|
| Никакого CI, пока польза не доказана | CI с Фазы 0 | Зелёный CI *и есть* часть сигнала читателю: проект живой и проверяемый |
| Никакого деплоя / K8s | Helm-чарт (Фаза 4) | Показывает, что система умеет разворачиваться не только на ноутбуке |
| Никакого auth | API-ключи | Демонстрирует OLTP-схему и rate limiting — иначе PostgreSQL в проекте нечем занять |
| Строй только для себя | Строится, чтобы это читали | Весь смысл — во внешней читаемости |

## Что остаётся неизменным

- **Узкий scope:** ровно три сервиса, никакого расползания фич. Список «Чего НЕ делать» соблюдается.
- **Никакого LLM внутри продукта:** сигнал этого проекта — данные и скорость, а не AI-фича,
  пришитая сбоку.
- **Сначала план, потом код.**
- **Каждое число измерено:** ни одна цифра производительности не попадает в документацию, если она
  не трассируется к воспроизводимому прогону бенчмарка.

## Последствия

- Репозиторий несёт больше инфраструктуры, чем нужно личному инструменту. Это цена читаемости, и
  она платится осознанно.
- Предохранитель от превращения этого в *настоящий* оверинжиниринг — список «Чего НЕ делать» и
  ограничение «ровно два рантайм-сервиса плюс консьюмер». Дисциплина scope остаётся; минимализм
  ради минимализма не применяется, когда работа артефакта — быть проверяемым.
